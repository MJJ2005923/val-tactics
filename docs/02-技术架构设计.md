# 02 — 技术架构设计

## 技术选型

| 层面 | 选型 | 版本 | 选型理由 |
|------|------|------|----------|
| 构建工具 | Vite | 6.x | 快速冷启动，HMR 极快 |
| UI 框架 | React | 19.x | 生态最大，社区资源丰富 |
| 类型系统 | TypeScript | 5.x | 类型安全，减少运行时错误 |
| 拖拽库 | @dnd-kit | latest | 轻量、现代、可定制、无障碍 |
| 本地存储 | Dexie.js | 4.x | IndexedDB 封装，API 简洁 |
| 样式方案 | CSS Modules | 内置 | 零依赖，天然隔离 |
| 部署 | Vercel | — | 免费，自动 HTTPS，CI/CD |

## 组件树

```
App
├── Navbar              # 顶部导航栏
│   ├── Logo
│   ├── MapSelector     # 地图选择下拉框
│   └── ActionButtons   # 导入/导出/模板管理按钮
├── MainArea
│   ├── AgentPanel      # 左侧英雄列表
│   │   ├── AgentSearch # 搜索框
│   │   ├── AgentItem[] # 英雄条目（可展开）
│   │   │   └── SkillIcon[] # 技能图标（可拖拽）
│   │   └── SkillDetail # 技能详情弹窗
│   └── CanvasArea      # 地图画布
│       ├── MapCanvas   # 底层地图渲染（Canvas）
│       └── MarkerLayer # 标记点层（SVG）
│           └── Marker[]# 技能标记（可拖动、点击）
├── Timeline            # 底部时间轴
│   ├── TimelineHeader  # 折叠/展开头
│   └── TimelineTrack  # 技能时间条
│       └── TimelineItem[] # 时间轴上的技能条目
└── TemplateManager     # 模板管理弹窗（模态框）
```

## 数据流

```
用户操作
  → React State (useState / useReducer)
    → 更新 UI
    → 必要时写入 IndexedDB（保存模板）

导入 JSON
  → 数据校验
    → 写入 IndexedDB
      → 更新 State → 刷新 UI
```

状态不跨组件共享时用本地 state，需要跨组件时通过 props 传递或 React Context。

## 性能策略

- 地图使用 Canvas 渲染，避免大量 DOM
- 技能标记使用 SVG overlay，数量可控（通常 ≤ 25 个/模板）
- 大图片使用 lazy loading
- IndexedDB 读写异步，不阻塞 UI
