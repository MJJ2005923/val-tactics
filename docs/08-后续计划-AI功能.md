# 后续计划：用户自助接入 AI

> 状态：规划中 | 优先级：低 | 预计工期：3-5 天

---

## 一、目标

让用户可以自己配置 AI API Key，在战术板中使用 AI 辅助功能。
**费用由用户自行承担，网站方零成本。**

---

## 二、架构设计

```
用户浏览器                    Cloudflare Worker               AI 平台
┌──────────┐     POST/api      ┌──────────┐   转发+Key       ┌──────────┐
│ 前端页面  │ ───────────────→ │ 代理中转  │ ───────────────→ │ Anthropic │
│          │                  │          │                  │ OpenAI    │
│ Key 存   │ ←─────────────── │ 无状态   │ ←─────────────── │ 等等      │
│ localStorage              │ 不存 Key │                  │          │
└──────────┘                  └──────────┘                  └──────────┘
```

**原理：**
1. 用户在网站设置页面填入自己的 API Key → 存浏览器 localStorage（不上传服务器）
2. 每次 AI 请求，前端把 Key + 问题一起发给 Worker
3. Worker 只做**纯转发**，不存日志、不存 Key、不做任何处理
4. Worker 返回 AI 结果给前端

**为什么需要 Worker？**
- AI 平台有 CORS 限制，浏览器无法直接调用
- Worker 充当一个中继站，绕开跨域限制

## 三、分阶段实施

### 阶段 A：基础设施（1 天）

**A1. 创建设置页面**
- 新建 `SettingsPanel` 组件
- API Key 输入框（密码遮蔽）
- 选择 AI 平台：Anthropic（Claude）/ OpenAI（GPT）
- Key 加密存储到 localStorage

**A2. 创建 Cloudflare Worker**
- 单一 API 端点 `/api/ai`
- 接收前端请求（含 Key + Prompt）
- 转发到对应 AI 平台
- 流式响应支持（SSE）

**A3. 模型选择**
- 支持切换模型：Claude Haiku / Sonnet / Opus
- 显示各模型价格参考

---

### 阶段 B：AI 战术助手（2-3 天）

**B1. 战术分析**
- 选中地图 → 已有技能布局 → AI 分析战术优劣
- 输出：盲点提示、技能配合建议、改进方案

**B2. 阵容推荐**
- 用户选择地图 + 攻防方
- AI 根据当前 meta 推荐 5 人阵容
- 自动填充到阵容槽

**B3. 自然语言放技能**
- 输入框：「在 A 点封两颗烟」
- AI 识别意图 → 自动放置对应技能到地图
- 支持：「铁臂闪 B 长」「欧门传送绕后」等

**B4. 战术描述生成**
- 一键总结当前战术布局
- 输出：可复制的文字战术描述
- 示例：「进攻方 5 人集结 B 长，Jett 先手进点，Omen 封烟断后路...」

---

### 阶段 C：进阶功能（2-3 天）

**C1. AI 对话面板**
- 侧边栏内嵌 AI 聊天窗口
- 上下文：当前地图 + 已放置技能
- 支持：文字输入 / 语音输入

**C2. 对手战术预判**
- 输入对手常用战术描述
- AI 分析破解方案
- 自动在地图上标注应对路线

**C3. 战术学习**
- 保存历史战术 → AI 分析规律
- 自动修正不合理布局
- 生成个性化战术建议

## 四、技术要点

### Worker 关键代码结构
```javascript
export default {
  async fetch(request) {
    const { apiKey, provider, model, messages } = await request.json()
    
    let response
    if (provider === 'anthropic') {
      response = await fetch('https://api.anthropic.com/v1/messages', {
        method: 'POST',
        headers: {
          'x-api-key': apiKey,
          'anthropic-version': '2023-06-01',
          'content-type': 'application/json',
        },
        body: JSON.stringify({ model, max_tokens: 1024, messages }),
      })
    }
    // OpenAI 同理...
    
    return new Response(response.body, {
      headers: { 'content-type': 'text/event-stream' },
    })
  }
}
```

### 前端请求示例
```typescript
const apiKey = localStorage.getItem('ai-api-key')
const provider = localStorage.getItem('ai-provider') || 'anthropic'

const response = await fetch('/api/ai', {
  method: 'POST',
  body: JSON.stringify({
    apiKey, provider,
    model: 'claude-haiku-4-5-20251001',
    messages: [
      { role: 'user', content: '帮我分析这张地图的B点防守阵型' }
    ]
  })
})
```

### 安全措施
- API Key 仅存 localStorage，不上传服务器
- Worker 不记录任何请求日志
- 设置页面可选「记住 Key」或「每次输入」
- 支持一键清除所有本地数据

### 技术架构

```
用户浏览器                     Cloudflare Workers          支付渠道
┌──────────────┐              ┌──────────────────┐        ┌──────────┐
│ 输入激活码    │ ──验证码──→  │ /api/verify-code  │ ────→  │ 预置码库  │

│              │              └──────────────────┘        └──────────┘

│              │              │ 转发 AI 请求       │
└──────────────┘              └──────────────────┘
```

### 防破解措施

因为是静态网站，前端付费逻辑可被绕过。需要 Worker 端做权限验证：

- **激活码验证在 Worker 端**，不在前端判断
- 验证成功后返回一个签名令牌，存 Worker KV
- 每次 AI 请求时 Worker 检查令牌有效性
- 令牌可设置有效期（月度/年度）

## 九、独立 AI 战术助手（远期愿景）

> 状态：远期构想 | 与战术板的 AI 功能并行规划

一个**独立于战术板的纯文字 AI 产品**，专注无畏契约战术分析与教学。不需要依赖拳头的图片、地图、特工头像，全靠 AI 知识和对话能力服务。

讨论一个游戏的知识不需要游戏公司授权。就像你可以写一本卖钱的篮球战术书，不需要 NBA 同意。AI 分析战术、教学、回答问题是**知识服务**，不是 IP 衍生品。

| 形态 | 说明 | 收费可行性 |
|------|------|-----------|
| 微信小程序 | 国内用户最方便 | ✅ 微信支付接入简单 |
| 网页版 | 现有网站扩展 | ✅ 爱发电 / 激活码 |
| 知识星球/付费社群 | AI + 人工结合 | ✅ 现成付费体系 |
| Discord/QQ 机器人 | 社群场景使用 | 中等 |

### 功能方向

**新手教练：**
- 「我是新手，玩 Jett 应该怎么打？」
- 「解释一下 Bind 这张图怎么打」

**战术分析：**

**练习计划：**
- 「给我一个每天 30 分钟的训练计划」

## 十二、AI 数据飞轮：用户使用 → 知识积累 → 越用越强

### 核心理念

不用自己造数据。AI 功能上线后，**用户的每一次提问、每一次纠错，都会自动喂养知识库**，形成正向循环：

```
用户使用               数据沉淀              知识进化
┌─────────┐         ┌──────────┐         ┌──────────┐
│ 问战术   │ ────→   │ 匿名问答对 │ ────→   │ 修正错误 │
│ AI回答   │         │          │         │ 补充新知 │
│ 👍/👎   │         │ 用户评分  │         │ 发现趋势 │
└─────────┘         └──────────┘         └──────────┘
     ↑                                        │
     └──────────── 越用越准 ←─────────────────┘
```

### 数据收集策略

| 数据类型 | 收集方式 | 隐私级别 | 知识价值 |
|------|------|------|------|
| AI 问答记录 | 自动匿名 | 🟢 安全 | ⭐⭐⭐⭐⭐ |
| 用户纠偏（👎） | 自动标记 | 🟢 安全 | ⭐⭐⭐⭐⭐ |
| 战术模板分享 | 用户主动 | 🟡 需同意 | ⭐⭐⭐⭐ |
| 技能使用频率 | 百度统计 | 🟢 安全 | ⭐⭐⭐ |
| 热门地图/特工 | 百度统计 | 🟢 安全 | ⭐⭐⭐ |
| API Key / 密码 | ❌ 绝不收集 | — | — |

```json
        { "content": "三闪Rush也很强，Jett+Phoenix+Reyna...", "score": 4.2, "from_ai": true }
```
