# 后续计划：用户自助接入 AI

> 状态：规划中 | 优先级：低 | 预计工期：3-5 天

---

## 一、目标

让用户可以自己配置 AI API Key，在战术板中使用 AI 辅助功能。
**费用由用户自行承担，网站方零成本。**

---

## 二、架构设计

```
用户浏览器                    Cloudflare Worker               AI 平台
┌──────────┐     POST/api      ┌──────────┐   转发+Key       ┌──────────┐
│ 前端页面  │ ───────────────→ │ 代理中转  │ ───────────────→ │ Anthropic │
│          │                  │          │                  │ OpenAI    │
│ Key 存   │ ←─────────────── │ 无状态   │ ←─────────────── │ 等等      │
│ localStorage              │ 不存 Key │                  │          │
└──────────┘                  └──────────┘                  └──────────┘
```

**原理：**
1. 用户在网站设置页面填入自己的 API Key → 存浏览器 localStorage（不上传服务器）
2. 每次 AI 请求，前端把 Key + 问题一起发给 Worker
3. Worker 只做**纯转发**，不存日志、不存 Key、不做任何处理
4. Worker 返回 AI 结果给前端

**为什么需要 Worker？**
- AI 平台有 CORS 限制，浏览器无法直接调用
- Worker 充当一个中继站，绕开跨域限制

## 三、分阶段实施

### 阶段 A：基础设施（1 天）

**A1. 创建设置页面**
- 新建 `SettingsPanel` 组件
- API Key 输入框（密码遮蔽）
- 选择 AI 平台：Anthropic（Claude）/ OpenAI（GPT）
- Key 加密存储到 localStorage

**A2. 创建 Cloudflare Worker**
- 单一 API 端点 `/api/ai`
- 接收前端请求（含 Key + Prompt）
- 转发到对应 AI 平台
- 流式响应支持（SSE）

**A3. 模型选择**
- 支持切换模型：Claude Haiku / Sonnet / Opus
- 显示各模型价格参考

---

### 阶段 B：AI 战术助手（2-3 天）

**B1. 战术分析**
- 选中地图 → 已有技能布局 → AI 分析战术优劣
- 输出：盲点提示、技能配合建议、改进方案

**B2. 阵容推荐**
- 用户选择地图 + 攻防方
- AI 根据当前 meta 推荐 5 人阵容
- 自动填充到阵容槽

**B3. 自然语言放技能**
- 输入框：「在 A 点封两颗烟」
- AI 识别意图 → 自动放置对应技能到地图
- 支持：「铁臂闪 B 长」「欧门传送绕后」等

**B4. 战术描述生成**
- 一键总结当前战术布局
- 输出：可复制的文字战术描述
- 示例：「进攻方 5 人集结 B 长，Jett 先手进点，Omen 封烟断后路...」

---

### 阶段 C：进阶功能（2-3 天）

**C1. AI 对话面板**
- 侧边栏内嵌 AI 聊天窗口
- 上下文：当前地图 + 已放置技能
- 支持：文字输入 / 语音输入

**C2. 对手战术预判**
- 输入对手常用战术描述
- AI 分析破解方案
- 自动在地图上标注应对路线

**C3. 战术学习**
- 保存历史战术 → AI 分析规律
- 自动修正不合理布局
- 生成个性化战术建议

## 四、技术要点

### Worker 关键代码结构
```javascript
export default {
  async fetch(request) {
    const { apiKey, provider, model, messages } = await request.json()
    
    let response
    if (provider === 'anthropic') {
      response = await fetch('https://api.anthropic.com/v1/messages', {
        method: 'POST',
        headers: {
          'x-api-key': apiKey,
          'anthropic-version': '2023-06-01',
          'content-type': 'application/json',
        },
        body: JSON.stringify({ model, max_tokens: 1024, messages }),
      })
    }
    // OpenAI 同理...
    
    return new Response(response.body, {
      headers: { 'content-type': 'text/event-stream' },
    })
  }
}
```

### 前端请求示例
```typescript
const apiKey = localStorage.getItem('ai-api-key')
const provider = localStorage.getItem('ai-provider') || 'anthropic'

const response = await fetch('/api/ai', {
  method: 'POST',
  body: JSON.stringify({
    apiKey, provider,
    model: 'claude-haiku-4-5-20251001',
    messages: [
      { role: 'user', content: '帮我分析这张地图的B点防守阵型' }
    ]
  })
})
```

### 安全措施
- API Key 仅存 localStorage，不上传服务器
- Worker 不记录任何请求日志
- 设置页面可选「记住 Key」或「每次输入」
- 支持一键清除所有本地数据

## 五、成本估算

### 网站运营方
| 项目 | 成本 |
|------|------|
| Cloudflare Worker | **免费**（每天 10 万次） |
| Cloudflare Pages | **免费** |
| AI API 费用 | **0 元**（用户自付） |

### 用户端（Anthropic 为例）
| 模型 | 每 1000 次查询约 |
|------|-----------------|
| Claude Haiku | ~$0.50（约 3.6 元） |
| Claude Sonnet | ~$3（约 22 元） |
| Claude Opus | ~$15（约 110 元） |

> 建议默认使用 Haiku，成本和速度最优，日常战术分析足够用
