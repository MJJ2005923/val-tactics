# 06 — 商业化规划：插件化架构

> 状态：规划完成，等待网站优化完毕后启动
> 创建日期：2026-06-03

---

## 一、背景与目标

当前"无畏契约战术板"代码约 **70% 是通用战术绘图引擎代码**，**30% 是 Riot Games IP 数据**。目标是将引擎与 IP 解耦，使引擎部分可以合法商业化。

### 核心原则

```
VALORANT 面板（免费）          TacticsBoard Engine（付费）
─────────────────────        ─────────────────────────
├── 地图/特工/技能（Riot IP）   ├── 绘图引擎
├── 拖拽布置技能               ├── 时间轴录制/回放
├── 编辑阵容                   ├── 模板管理
└── 引擎插槽 🧩               ├── 导入/导出
                              ├── 撤销/重做
                              ├── 云端同步（Pro）
                              └── 团队协作（Pro）
```

**引擎代码不含任何 Riot IP，通用到可应用于任何游戏或场景。**

## 二、法律风险分析

| 做法 | 风险 | 说明 |
|------|------|------|
| 网站整体收费 | 🔴 极高 | 直接利用 Riot IP 盈利 |
| 免费基础 + 付费高级功能 | 🟠 中高 | 付费功能依赖 VALORANT 数据运行 |
| **通用引擎 + 互不导流** | **🟡 中低** | 引擎干净，面板免费，两不相干 |
| 纯打赏/赞助 | 🟢 低 | 标准粉丝项目做法 |
| 拿到 Riot 官方授权 | 🟢 零 | 需要正式申请 |

> ⚠️ 做不到 100% 零风险。但只要引擎本身干净、面板永远免费，即使面板被要求下架，引擎收入不受影响。

### 3.2 引擎导入流程

```
用户点击"导入引擎"
    → 弹出 License Key 输入框
    → 调用验证 API (POST api.tacticsboard.io/verify)
    → 验证成功 → 返回引擎 JS CDN URL
    → 动态加载引擎 (import() 或 script 注入)
    → 引擎初始化 mount(container, boardConfig)
    → 面板即时升级，高级功能解锁
```

### 3.3 License Key 机制

- 格式：`TB-{产品码}-{用户ID哈希}-{校验码}`
- 验证：在线验证 + 本地缓存 24 小时
- 防滥用：同 Key 最多 3 设备同时在线

## 五、实施路线图

### 第 1 步：引擎核心独立（2-3 周）

把 engine 代码抽成独立 JS bundle，完全脱离 VALORANT 数据可运行。

- 定义 `BoardConfig`、`Entity`、`Ability`、`BoardMap` 等泛型接口
- 所有组件改为接收数据接口，不硬编码 agents/maps
- 用 Vite library mode 构建引擎为独立 JS 文件
- 创建虚构 Demo 数据（足球/FPS/消防）验证通用性

**产物**：`tacticsboard-engine-v1.0.0.js`

### 第 2 步：面板改造（1-2 周）

当前 VALORANT 面板变为"免费查看器 + 引擎插槽"。

- 引擎代码从面板剥离，只保留数据层和免费 UI
- 添加引擎插槽 UI + License Key 验证 + 动态加载
- 加入 Riot 粉丝项目声明 + 自愿打赏入口

### 第 3 步：验证 API + CDN（1-2 周）

- Cloudflare Worker：License Key verify API
- Cloudflare R2：引擎 JS 文件 CDN 存储
- Key 生成 + 数据库 + 防滥用

### 第 4 步：引擎官网 + 购买流程（1-2 周）

- 引擎官网（tacticsboard.io），绝不出现 VALORANT
- 定价页 + 虚构 Demo 演示
- Lemon Squeezy 支付集成（支付宝/微信）
- 购买 → Key → 安装指引

### 第 5 步：Pro 功能（3-4 周）

- 用户认证（Supabase Auth）
- 云端同步（Workers + KV/D1）
- 版本历史 + 高清导出 + 团队协作

### 第 6 步：生态扩展（长期）

- 面板开发规范文档 + 开源模板 + 第三方开发者
